# Qualitative regression benchmarks

这些案例用于固定 Skill 的关键判断边界，防止后续版本重新退化为：

- 只会删除网文式名字；
- 把“像真人”误当成主角名的充分条件；
- 为所有人物强行制造记忆点；
- 只评价法定姓名，不处理乳名、外号、法号和关系称呼；
- 用自动分数伪装文学判断。

`regression_cases.json` 只包含匿名化、合成案例。它验证决策类型和理由结构，不声称脚本能够判断名字的文学质量。

新增案例应满足：

1. 对应一个清楚的失败模式；
2. 不包含私人剧本或真实个人信息；
3. 说明应保留、淘汰、加强称呼还是改变命名制度；
4. 理由来自人物生活、声音、关系或剧情占有，而不是字义好恶。
