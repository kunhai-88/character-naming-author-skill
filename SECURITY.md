# Security Policy

## Supported versions

安全修复仅保证进入最新发布版本。请先确认问题在 `main` 或最新 Release 中仍可复现。

## Reporting a vulnerability

请不要为安全问题创建公开 Issue。使用 GitHub 仓库的 **Report a vulnerability** 私密报告入口，并提供：

- 受影响文件和版本；
- 最小复现步骤；
- 潜在影响；
- 建议修复（如有）。

维护者会在 7 天内确认收到报告。修复发布前，请勿公开利用细节。

## Privacy issues

意外提交的私人剧本、个人信息、本机路径或凭据也按安全问题处理。请在报告中标明需要从 Git 历史中清除的具体内容。
