# 匿名化快速示例

## 输入材料

> 现实家庭短剧。女主 42 岁，1984 年出生于县城，母亲是供销社职工，父亲是货车司机。她经营一家裁缝店，正在离婚。现名“沈知微”，丈夫“顾沉舟”，女儿“顾安然”。请审查整套姓名，不要为了展示工作量全部换掉。

## 正确处理顺序

1. 先判断三个人的出生年代与取名者；
2. 识别夫妻是否本来同姓、孩子随谁姓；
3. 把名字放进母亲、丈夫、客户和办事人员的称呼中；
4. 隐藏“知微、沉舟、安然”的字义解释再做盲审；
5. 只改不通过的人名。

## 示例结论

“沈知微 / 顾沉舟 / 顾安然”作为一组出现时，类型化程度过高：三个人都像编剧根据人物功能安排的精致三字名。问题不是某个字被禁止，而是整个家庭的命名过度整齐。

在没有具体地域姓名资料时，不应假装给出唯一终稿。可以先建立姓名带宽：

- 女主：1980 年代县城家庭常见的朴素双字或单字名；
- 丈夫：同代普通男性名，不预告善恶；
- 女儿：根据实际出生年份确定，而不是沿用父母一代审美。

候选必须在以下对白中通过试演：

```text
母亲：你先回来吃饭，别一个人扛。
丈夫：我们内部还没谈完。
客户：老师，我的衣服还能改吗？
工作人员：请本人确认签名。
```

这个示例刻意不提供“标准答案”。Skill 的目标不是替换一批漂亮字，而是阻止缺少出生背景时的虚假精确。
